# FlowAI v2 Features Plan

**Goal:** Add inbound webhook triggers, manual run button, and workflow templates.

**Architecture:** Extend the existing schema with a `triggerType` field (cron | webhook). Inbound webhooks get a dedicated route `/hooks/:id`. Manual run reuses the executor pipeline. Templates are a static list on the frontend.

---

## Feature 1: Manual "Run Now"

### Task 1: Backend — POST /api/workflows/:id/run

**Files:**
- Modify: `server/routes/workflows.js`
- Modify: `server/services/scheduler.js` (extract execution logic into reusable function)

Add `executeWorkflowNow(workflow, prisma)` to scheduler.js that runs a workflow's action immediately and logs the result. Add `POST /api/workflows/:id/run` route that calls it.

### Task 2: Frontend — Run Now button on Dashboard

**Files:**
- Modify: `client/src/lib/api.js` (add `runWorkflow(id)`)
- Modify: `client/src/pages/Dashboard.jsx` (add Run button per row)

Add a "run" button next to each workflow on the Dashboard. On click, calls the API, shows brief "running..." feedback.

---

## Feature 2: Inbound Webhook Triggers

### Task 3: Schema — add triggerType and webhookToken

**Files:**
- Modify: `server/prisma/schema.prisma`

Add `triggerType String @default("cron") @map("trigger_type")` and `webhookToken String? @unique @map("webhook_token")` to Workflow model. Run `prisma db push`.

### Task 4: Backend — inbound webhook route

**Files:**
- Create: `server/routes/hooks.js`
- Modify: `server/index.js` (mount hooks route)

`POST /hooks/:token` — look up workflow by webhookToken, execute its action, log the result. Pass the request body as `triggerPayload` so the action can reference it.

### Task 5: LLM parser — support trigger-based workflows

**Files:**
- Modify: `server/services/llm-parser.js`

Update the system prompt to support a new `trigger` field: `"cron"` (with cron expression) or `"webhook"` (no cron needed). When trigger is webhook, return `"cron": null`.

### Task 6: Workflow creation — handle webhook type

**Files:**
- Modify: `server/routes/workflows.js`

When creating a workflow with `trigger: "webhook"`, generate a UUID webhookToken, save it, skip cron registration. Return the webhook URL in the response.

### Task 7: Frontend — show webhook URL + handle trigger type

**Files:**
- Modify: `client/src/pages/Dashboard.jsx`
- Modify: `client/src/pages/Home.jsx`

Dashboard: for webhook-triggered workflows, show the webhook URL instead of cron expression. Home: after confirming a webhook workflow, show the URL to copy.

## Feature 3: Workflow Templates

### Task 8: Frontend — template picker on Home page

**Files:**
- Modify: `client/src/pages/Home.jsx`

Add 5-6 preset templates below the input field. Clicking one fills the input with that text. Templates: Discord standup reminder, hourly health check, weekly email report, webhook forwarder, daily Discord motivational message.
