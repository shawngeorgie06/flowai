# FlowAI Frontend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a three-view React frontend (Home, Dashboard, Logs) that connects to the existing FlowAI Express backend.

**Architecture:** Vite + React + Tailwind. React Router for three routes. Shared API module for all backend calls. Socket.io client for real-time log streaming. No state management library — local state and prop drilling only (three views, no shared state needed).

**Tech Stack:** React 18, Vite, Tailwind CSS v3, React Router v6, socket.io-client

### Task 2: API and Socket.io Client Modules

**Files:**
- Create: `client/src/lib/api.js`
- Create: `client/src/lib/socket.js`

- [ ] **Step 1: Create api.js**

```js
const BASE = '/api/workflows';

export async function parseWorkflow(input) {
  const res = await fetch(BASE, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ input }),
  });
  return res.json();
}

export async function confirmWorkflow(input) {
  const res = await fetch(BASE, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ input, confirm: true }),
  });
  return res.json();
}

export async function getWorkflows() {
  const res = await fetch(BASE);
  return res.json();
}

export async function toggleWorkflow(id) {
  const res = await fetch(`${BASE}/${id}/toggle`, { method: 'PATCH' });
  return res.json();
}

export async function deleteWorkflow(id) {
  await fetch(`${BASE}/${id}`, { method: 'DELETE' });
}

export async function getWorkflowLogs(id) {
  const res = await fetch(`${BASE}/${id}/logs`);
  return res.json();
}
```

- [ ] **Step 2: Create socket.js**

```js
import { io } from 'socket.io-client';

export default socket;
```
